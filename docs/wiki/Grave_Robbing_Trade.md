# Grave Robbing Trade

## Overview
The grave robbing trade is a custom `HarvestSystem` in `Server.Engines.Harvest`. Players use a `GraveShovel` `Item`, target a grave/tombstone tile, and the system rolls Forensics-based harvest resources, optional ML relic bonus resources, criminal-witness side effects, awakened undead spawns, and an optional `GraveChest`.

There are no custom `CommandSystem.Register` commands, `[Usage]` attributes, `Gump` classes, `NetState` packet handlers, or XMLSpawner attachments in the traced grave robbing scripts. Normal entry is item double-click targeting. Administration uses standard RunUO construction such as `[add GraveShovel]`, `[add Bones]`, or targeted construction of `GraveChest` if needed.

## Player Entry Flow
1. The player double-clicks a `GraveShovel` in their backpack or equipped layer.
2. `BaseHarvestTool.OnDoubleClick` calls `HarvestSystem.BeginHarvesting(from, this)`.
3. The shared harvest system either assigns a `HarvestTarget` directly or sends a captcha first, depending on `MyServerSettings.AllowMacroResources()`.
4. The player targets a static, static target, or land target.
5. `GraveRobbing` accepts only tile IDs listed in `m_GraveTiles`.
6. The target must be on a non-internal map and within `MaxRange = 1`.
7. The player cannot dig while mounted.
8. The player cannot dig while body-modded into a non-human body when `RaceID < 1`.
9. The shared harvest timer plays action `11`, sound `0x125` or `0x126`, then resolves the resource and side effects.

| Effect sound delay | `0.9` seconds. |

The shared `HarvestBank` consumes the delivered item amount. Stackable results receive `ConsumedPerHarvest` or the Isles of Dread amount; non-stackable results keep their constructed item amount. When an `IUsesRemaining` tool succeeds, the shared harvest code decrements uses and deletes the tool at zero uses.

## Harvest Side Effects
Side effects run when the selected `HarvestResource` is the vein primary resource. If the resource has already fallen back to `Bones`, this side-effect block does not run. The code does not check whether the shared harvest system actually awarded the primary item, so failed primary-resource skill rolls can still trigger these side effects.

| Effect | Code behavior |
| --- | --- |
| Karma loss | If `from.Karma > -2459`, awards `-50` karma. |
| Witness roll | `Utility.RandomMinMax(1, 100) < 3`, which is a 2% roll. |
| Stealth escape | If Hiding is at least `30.0`, the player receives a stealth message and then `CheckSkill(Stealth, 0, 100)` can prevent being caught. |
| Criminal report | If caught, sends overhead and chat messages, sets `from.Criminal = true`, and removes disguises through `DisguiseTimers.RemoveDisguise(from)`. |
| ML reveal | `OnHarvestStarted` calls `from.RevealingAction()` when `Core.ML` is enabled. |

## Awakened Undead
The system always constructs an undead `BaseCreature` candidate, then rolls whether it actually wakes up.

| Rule | Value |
| --- | --- |
| Spiritualism divisor | `nSpiritualism = (int)(Spiritualism.Value / 10)`. |
| Spawn condition | `Utility.Random(100) > (nSpiritualism + 85)`. |
| Effective chance | `max(0, 14 - nSpiritualism)%`. |
| Skill gain check | `from.CheckSkill(Spiritualism, 0, 100)` only runs when an undead spawns. |
| Placement | Tries eight offsets around the player, first at player Z, then average Z; if all fail, moves the Mobile to the player's current location. |
| Spawn state | Calls `OnBeforeSpawn`, sets `Home`, `RangeHome = 5`, appends ` [Awakened]` to `Title`, moves to world, says one random phrase, sets `ControlSlots = 666`, and sets `Combatant = from`. |

| Spawn Mobile | Selection chance |
| --- | ---: |
| `Shade` | `10%` |
| `Zombie`, `Skeleton`, `Ghoul`, `Spectre`, `Wraith`, `BoneKnight`, `BoneMagi`, `Ghostly`, `Lich`, `LichLord`, `Mummy`, `RottingCorpse`, `SkeletalKnight`, `SkeletalWizard`, `SkeletalMage`, `Phantom`, `Vampire`, `Bodak` | `5%` each |

## Grave Chest Roll
After the undead roll, the system attempts a grave chest roll. This is part of `OnHarvestFinished` and is gated by the primary-resource match, not by successful item delivery.

```text
digger = (int)(Forensics.Value / 10)
chest roll succeeds when (2 + digger) > Utility.Random(100)
then CheckSkill(Forensics, 0, 125) must also succeed
```

The raw chance before the `CheckSkill` gate is `(2 + digger)%`. Examples:

| Forensics value | `digger` | Raw chest-roll chance |
| ---: | ---: | ---: |
| `0.0..9.9` | `0` | `2%` |
| `50.0..59.9` | `5` | `7%` |
| `100.0..109.9` | `10` | `12%` |
| `120.0..129.9` | `12` | `14%` |

The chest level is selected by creating a default `GraveChest(6, from)` and then switching on `Utility.Random(10 + digger)`.

| Random result | Chest level |
| --- | ---: |
| `0..5` | `1` |
| `6..10` | `2` |
| `11..14` | `3` |
| `15..17` | `4` |
| `18..19` | `5` |
| `20+` | `6` via the initial default instance |

A successful chest is moved to the player's current `loc` and `map`, and the player receives `you dig up a graveyard chest.`

## Grave Chest Construction
`GraveChest` inherits `SkullChest`.

| Construction step | Behavior |
| --- | --- |
| Name | Sets `Name = "graveyard chest"`. |
| Fill | Calls `ContainerFunctions.FillTheContainer(level, this, digger)`. |
| Luck bonus | If `GetPlayerInfo.LuckyPlayer(digger.Luck)` returns true, fills the chest a second time. |
| Locking | Calls `ContainerFunctions.LockTheContainer(level, this, 1)`. |
| Hue | Randomly selects one of twenty hard-coded hues. |
| Visual form | 25% casket, 25% sarcophagus, 25% coffin, 25% default chest. |
| Owner text | `ContainerOwner = ContainerFunctions.GetOwner(sBox)`. |
| Digger text | `ContainerDigger = digger.Name`. |

`LockTheContainer(level, box, 1)` can set one of four trap types or no trap. For normal grave chests, each of dart, explosion, magic, and poison traps has a 1-in-9 chance; no trap has a 5-in-9 chance. Trap power is `(level * Utility.RandomMinMax(20, 30)) + Utility.RandomMinMax(1, 10)`. Locking itself is a 1-in-3 chance, with `LockLevel = 1 + (level * 10)`, `MaxLockLevel = LockLevel + 20`, and `RequiredSkill = LockLevel`.

`FillTheContainer` is the shared dungeon-container loot helper. It caps input levels above 12, computes money from `(level + 1) * Utility.RandomMinMax(40, 160)` scaled by `MyServerSettings.GetGoldCutRate()`, drops currency or map-specific substitutes, and marks `GraveChest` as `IgnoreThisFiller = 1`, which skips several generic filler branches used by ordinary dungeon containers.

## Known Issues
| Issue | Impact |
| --- | --- |
| Primary-resource side effects do not check whether the shared harvest actually awarded an item. | A failed Forensics harvest can still apply karma loss, criminal witnessing, awakened undead, and chest rolls if the selected resource did not fall back to `Bones`. |
| `OnHarvestFinished` creates an undead `BaseCreature` before the Spiritualism spawn roll and does not delete it when the roll fails. | Failed spawn rolls can leave unplaced, undeleted Mobile instances. |
| The whole reward side-effect block is wrapped in `catch { }`. | Exceptions during karma, criminal reporting, undead creation, or chest creation are silently hidden. |
| The comment says the undead roll is a 10% chance, but the compiled formula is `max(0, 14 - (int)(Spiritualism / 10))%`. | Admin expectations will be wrong, especially at low or high Spiritualism values. |
| The Hiding/Stealth witness branch sends the "avoided being seen" message before the Stealth skill check. | A player can see successful-avoidance text and then still be reported as a criminal if the Stealth check fails. |
| `m_GraveTiles` repeats `0x5170` and does not include `0x5179` in the otherwise contiguous `0x5164..0x5183` block. | One intended grave/tombstone art tile may be missing from the valid target list. |
| `GraveChest(int level, Mobile digger)` dereferences `digger.Luck` and `digger.Name` with no null guard. | Unexpected null construction paths can throw during chest creation. |
| The shared harvest bonus failure branch deletes `item` instead of `bonusItem`. | Rare bonus-placement failures can delete the primary harvest result while leaving the failed bonus object unhandled. |
